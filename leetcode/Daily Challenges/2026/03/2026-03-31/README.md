## Instructions
You are given two strings, `str1` and `str2`, of lengths `n` and `m`, respectively.

A string `word` of length `n + m - 1` is defined to be **generated** by `str1` and `str2` if it satisfies the following conditions for **each** index `0 <= i <= n - 1`:

- If `str1[i] == 'T'`, the **substring** of `word` with size `m` starting at index `i` is **equal** to `str2`, i.e., `word[i..(i + m - 1)] == str2`.
- If `str1[i] == 'F'`, the **substring** of `word` with size `m` starting at index `i` is **not equal** to `str2`, i.e., `word[i..(i + m - 1)] != str2`.

Return the **lexicographically smallest** possible string that can be **generated** by `str1` and `str2`. If no string can be generated, return an empty string `""`.

**Example 1:**
**Input:** str1 = "TFTF", str2 = "ab"
**Output:** "ababa"
**Explanation:**
#### The table below represents the string `"ababa"`

|Index|T/F|Substring of length `m`|
|---|---|---|
|0|`'T'`|"ab"|
|1|`'F'`|"ba"|
|2|`'T'`|"ab"|
|3|`'F'`|"ba"|

The strings `"ababa"` and `"ababb"` can be generated by `str1` and `str2`.

Return `"ababa"` since it is the lexicographically smaller string.

**Example 2:**
**Input:** str1 = "TFTF", str2 = "abc"
**Output:** ""
**Explanation:**
No string that satisfies the conditions can be generated.

**Example 3:**
**Input:** str1 = "F", str2 = "d"
**Output:** "a"

**Constraints:**

- `1 <= n == str1.length <= 104`
- `1 <= m == str2.length <= 500`
- `str1` consists only of `'T'` or `'F'`.
- `str2` consists only of lowercase English characters.

## My Thoughts

This problem is easiest to handle in two phases:

- force every `'T'` constraint
- then make sure every `'F'` constraint is broken while keeping the string lexicographically smallest

Phase 1: place required matches

- for every index `i` where `str1[i] == 'T'`, the window `word[i..i+m-1]` must exactly equal `str2`
- write those characters into the answer
- if two required windows conflict, return `""`

Phase 2: fill the rest minimally

- initialize all still-unfixed positions to `'a'` to keep the string lexicographically smallest

Phase 3: fix invalid `'F'` windows

- if an `'F'` window already differs from `str2`, it is fine
- if it exactly matches `str2`, change the rightmost non-fixed character in that window
- using the rightmost free position preserves lexicographic minimality as much as possible
- if the window is fully fixed and still equals `str2`, return `""`

This greedy approach works because all hard equality requirements are locked in first, then each forbidden window is broken with the smallest possible damage.

Time complexity: `O(n * m)`  
Space complexity: `O(n + m)`

## What I Learned

- Constraint problems often become simpler when “must match” requirements are applied before “must differ” requirements.
- Lexicographically smallest construction usually means delaying changes as far to the right as possible.
- Marking fixed positions is useful when later edits are only allowed on flexible characters.
- A good pattern for forbidden-window problems is: build the smallest candidate first, then minimally break any invalid exact matches.
